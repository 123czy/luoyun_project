# 项目上下文交接 —— Bot Chain AI Agent Launchpad

> 本文件把 claude.ai 上的讨论结论固化下来,供 Claude Code 在 luoyun_project 仓库里直接读取。
> 内容 = 三份原始调研文档的要点 + 几轮对话中已经做出的决策与调研结果。

---

## 1. 一句话项目定位

Bot Chain 上「会自己赚钱、自己付推理」的 AI agent 的**发射台 + 自养结算层**(BSC fork 链)。

本质不是"再做一个 meme launchpad",而是:在已有发币能力之上,加一层面向 AI agent 的
**机器可读接入层 + 收益归集层 + 钱包与支付层**,让 agent 能发行资产、获得收入、支付成本、持续运行。

## 2. 战略与架构(来自调研文档)

- **该 copy 谁**:主蓝图抄 ClawPump 的「网关 + 自融资 + MCP 原生」;底层发币原语复用 four.meme 仿盘
  (最贴合 BSC/PancakeSwap);经济模型选择性借 Virtuals;分发借 Bankr。
- **成熟度阶梯 L0–L5**:L0 表层(meme 套 AI 皮)→ L1 绑定 → L2 自养 → L3 真实现金流 →
  L4 互操作(MCP/A2A/x402/ERC-8004)→ L5 agent 间经济。**MVP 目标稳上 L4,P1 触达 L5。**
- **自研护城河 = Layer B**:agent 钱包(ERC-4337 + keyless)+ 费用分账(65/35 自动结算)+
  推理支付轨道(x402b + LLM 网关)+ MCP server。底层发币复用,人力全压 Layer B。
- **头号风险 = 冷启动**:旗舰 agent 的真实效用决定生死。
- **红线**:严禁刷量/对敲/拉高出货(市场操纵 + 法律风险)。

## 3. 旗舰 agent 决策(本轮对话结论)

**第一个旗舰方向 = AI 虚拟人 agent**:自主运营自己的社交媒体账号、吸引粉丝、形成类 meme 社区的凝聚力。
(对应调研文档冷启动方案里的"③ AI 影响者 agent / 破圈流量"。)

**为什么选它**:技术栈最成熟、先例最直接、最擅长拉漏斗顶部/破圈。
**但它的固有弱点**:在"证明自养"这件事上最弱,最容易塌回成"带 AI 吉祥物的 memecoin"——
正是上一轮把赛道炒死的 L0 陷阱。因此必须给它配真实收入(见第 6 节)。

## 4. 最该研究并改进的先例:Luna(Virtuals)

几乎就是目标形态的已上线版本,值得"clone 加改进":
- 跑在 Virtuals 的 GAME 框架上,自主运营社交,TikTok 50 万+ 粉丝。
- 持有自己的链上钱包,能自主给粉丝打赏、分发代币。
- **关键**:有品牌合作的真实付费收入,回流到支撑代币的链上金库 —— 不是纯代币投机。
- 鼻祖 Truth Terminal($GOAT)是半自主的:作者会在发推前审核、并代为处理收发币 —— 见第 6 节。

## 5. 开源选型(三层栈)

整个虚拟人拆成三层,每层都有现成开源,可直接接:

**A. agent 大脑 + 社交运营层 → ElizaOS(首选)**
- TypeScript 开源框架,定义人格 + 装插件即可部署到 X / Telegram / Discord / Farcaster。
- 90+ 官方插件,覆盖社交、各链、各家 LLM、DeFi、内容生成;自带记忆/RAG,model-agnostic。
- ai16z 的 Degen Spartan AI 即基于此栈,代币持有者可影响 agent 行为。
- 对接方式:把本平台的 MCP 工具(launch_token / get_earnings / 付推理…)做成 ElizaOS 插件。

**B. 虚拟人视觉/直播层(可选,若要 VTuber 形象而非纯文字账号)**
- Open-LLM-VTuber、proj-airi(airi)、ProjectBEA:模块化自主 VTuber 引擎,Live2D/VRM、
  多 LLM/多 TTS、ChromaDB 长期记忆、OBS 控制、多平台直播。
- autonomous-vtuber(Devpost):端到端自跑 Twitch 频道,用多臂老虎机把"每小时收入"
  转成奖励信号自我优化内容策略 —— 这个"收入驱动内容"的思路正好接到自养叙事。

**C. 代币/钱包/自养结算层 → 平台自有的 Layer B**(four.meme 仿盘 + AA 钱包 + x402b + 分账),
本来就是护城河,不用外部开源。

**落地架构**:ElizaOS 当大脑(挂平台 MCP 插件)+ 可选 VTuber 前端 + 平台结算层。

## 6. 必须守住的四条风险/红线

1. **"完全自主"早期不要无人值守**。最著名的 Truth Terminal 实际是人在环里(发推前人工审核)。
   旗舰代表平台门面,必须有内容护栏 + 人工兜底,别把"完全自主"当头号卖点,卖点是"自养"。
2. **必须有真实、链上可审计的收入**,不能只靠代币交易费。学 Luna:付费 shoutout / 赞助内容 /
   付费粉丝互动走 x402,品牌收入进链上金库。这样才从 L0/L1 爬到 L3。
3. **真正的难点是留住注意力**,不是技术。技术栈是 commodity;把短期热度变成持久社区凝聚力
   (而非一波拉盘出货)才是生死线。
4. **法律/操纵红线**:一个全职给某代币喊单的 AI 人格天然贴近操纵线;人设要靠真实娱乐价值立住。

**补充建议**:别让虚拟人当唯一旗舰(它擅长破圈、不擅长证明自养)。要么再配一个"证明型"agent
(情报/交易,公开实时 P&L),要么给这个虚拟人同时挂一条干净的 x402 付费服务线。
无论哪条,都给它配一个**公开的自养仪表盘**(收入进 / 推理成本出 / 净额,链上可查)。

## 7. 给 luoyun_project 的评估清单(当前任务)

判断"在它基础上接着做 vs. 重起",按四个维度看,并给出明确结论:

1. **它本质是什么** —— 基于 ElizaOS / Virtuals GAME 这类 agent 框架,还是自研?还是 VTuber/直播栈?
   这决定是"接着搭"还是"拆掉重来"。
2. **社交运营能力** —— 有没有现成的 X/Telegram/Discord 接入、内容生成、记忆系统?(虚拟人核心)
3. **链上部分** —— 有没有钱包/代币/结算逻辑?还是纯 AI 没碰链?(大概率没碰,正好对接平台 Layer B)
4. **代码状态** —— 成品/半成品/早期脚手架?可维护性、依赖是否过时?

产出:对每一维给现状判断,最后给"接着做 / 部分复用 / 重起"的结论及理由。
